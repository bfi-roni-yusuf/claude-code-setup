# Generate Seed Data

Generate realistic SQL INSERT statements from JPA entity definitions.

## Usage

`/generate-seed-data <entity-name-or-path> [--fk col=val ...] [--rows N] [--execute]`

- `entity-name-or-path`: Class name (e.g., `SurveyorAssignmentCapitalDetail`) or file path
- `--fk`: Foreign key column=value pairs. Repeat for multiple. If omitted, see "Missing FK" below.
- `--rows`: Number of rows to generate (default: 1)
- `--execute`: Execute the SQL against local DB instead of printing raw SQL

## Process

1. **Read the entity** — Find via `Glob **/entity/**/<Name>.java`. If no match is found, try broader search with `Grep` for the class name. If still not found, report the error and stop.

   **Missing FK handling:** After reading the entity, identify ALL `@JoinColumn` fields. If `--fk` was not provided for any of them, list all missing FKs in a single prompt:
   ```
   This entity requires the following FK values:
   - `assignment_id` → SurveyorAssignment (missing)
   - `application_id` → Application (missing)
   Please provide values for all missing FKs.
   ```
   Only list the ones not already supplied via `--fk`. Do NOT generate SQL with placeholder or made-up FK values.

2. **Read BaseEntity** (`entity/common/BaseEntity.java`) — inherited columns:
   - `version` → `0`
   - `deleted` → `false`
   - `created_at` → `NOW()`
   - `created_by` → `'system'` (DB default per migration V1_0_202208081500)
   - `updated_at` → `NOW()`
   - `updated_by` → `'system'`
   - `deleted_at` → omit (nullable, defaults to NULL)

3. **Read ALL referenced types** — this is critical, do NOT guess:
   - `@Enumerated` fields → find and read the enum class for actual constant names. Use the enum constant name as the SQL string value (e.g., `'PRODUCTIVE'`, not the constructor parameter).
   - `@Type(JsonBinaryType.class)` fields → find and read the referenced class (e.g., `Document`). Check `@JsonProperty` annotations for actual JSON key names — they often differ from Java field names (e.g., `fileName` → `"file_name"`).
   - `@OneToOne` / `@ManyToOne` → FK column, use value from `--fk`

4. **Derive table name**:
   - `@Table(name = "x")` → use `x`
   - `@Entity(name = "x")` where name is already snake_case → use `x`
   - Neither → convert class CamelCase → snake_case (e.g., `MyEntity` → `my_entity`)

5. **Map columns**:
   - Skip `@Id` with `@GeneratedValue` — auto-generated by DB
   - `@JoinColumn(name = "x")` → column name is `x`, value from `--fk`
   - `@Column(name = "x")` → use explicit `x`
   - Default → convert Java camelCase field name → snake_case

6. **Generate values** per type mapping below

7. **Output or execute**:
   - Default → print raw SQL to chat
   - `--execute` → run via psql (see CLAUDE.md "Local Database Access" for connection string)

## Type → Value Mapping

| Java Type | SQL Value |
|---|---|
| `BigDecimal` | IDR-scale amounts (millions–billions). Respect `precision`/`scale` from `@Column`. |
| `String` (code) | Realistic code matching field context (e.g., `'S1'`, `'OWN'`) |
| `String` (name) | Indonesian description matching the code (e.g., `'Sarjana'`, `'Milik Sendiri'`) |
| `String` (NPWP) | 15-digit format: `'XXXXXXXXXXXXXXX'` |
| `String` (KTP/NIK) | 16-digit NIK: `'32XXXXXXXXXXXXXX'` |
| `Integer` | month → 1–12, year → 2024–2026, count → 0–5 |
| `Long` (FK) | From `--fk` argument |
| `Long` (other) | Sensible value for context (e.g., `currentPastDueDay` → 0) |
| `Boolean` | Contextual; `false` for `deleted` |
| `LocalDate` | Recent date: `'2025-01-15'` |
| `ZonedDateTime` | `NOW()` for audit fields |
| `UUID` | `gen_random_uuid()` or a literal UUID string |
| `List<T>` (JSON) | JSON array with 1–2 items. MUST use field names from actual class `T`. |
| Enum (`@Enumerated(STRING)`) | Actual constant from the enum class. MUST read the enum first. |

## Financial Data Coherence

When the entity represents financial data (balance sheet / income statement):
- **Assets = Liabilities + Equity**
- Summary fields must equal the sum of their detail fields
- `gross_profit = operating_income - cost_of_goods_sold`
- `current_year_profit ≈ profit_before_tax × 0.75` (25% tax)
- For multiple rows: show growth trend (5–10% per period)

## Multiple Rows (`--rows > 1`)

Vary across rows: names, codes, dates, amounts (±10–20%), different enum values.

## Common Mistakes

| Mistake | Fix |
|---|---|
| Including auto-gen `id` | Skip any `@Id @GeneratedValue` column |
| camelCase column names | Always convert to snake_case |
| Missing BaseEntity fields | Always include `version`, `deleted`, `created_at/by`, `updated_at/by` |
| Guessing enum values | Read the actual enum source file first — values MUST exist |
| Guessing JSON structure | Read the referenced type class — field names MUST match |
| Wrong FK column name | Read `@JoinColumn(name = "...")` — don't guess from Java field name |
| Including `deleted_at` | Omit — nullable column, defaults to NULL |
